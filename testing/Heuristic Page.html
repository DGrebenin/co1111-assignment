Heuristic Page

User Acceptance Plan:

The web app was given to some members of team two and also to common people, for "black box testing" in order to identify and fix mistakes. The common suggestion from all people who tested the game is to improve the design when the quiz starts. Moreover, the common people that have no relation with the Uclan, cannot answer some questions. Despite of the above, they did not identify any usability issues.

Usability Report:

Heuristic Details Exist:
1. Visibility of system status Server error messages are displayed, for instance, if the player's device has not have cameras, an alert message is displayed ✔
2. Match between system and the real world We are using universal icons, such as the QR code icon, which are commonly understood. ✔
3. User control and freedom If the users scan a QR code, automatically is displayed the decoded message ✔
4. Consistency and standards We are using the same colors and layout in all pages. ✔
5. Error prevention The possible answers are displayed according to the type of the question. For example, MCQ answers are shown as a multiple choice form and BOOLEAN answers are shown using True/False. ✔
6. Recognition rather than recall When a new question is displayed, its contents and answer are shown at the same location every time which is what is expected from the player. ✔
7. Flexibility and efficiency of use The users can exit the game and they can continue whenever they want from the same point they left. ✔
8. Aesthetic and minimalist design The web app does not have intensive colors or huge images that distract the players from the game. ✔
9. Help users recognize, diagnose, and recover from errors If the user enters a name that already exists in a game, a warning message is displayed and the user can try entering a different name. ✔
10. Help and documentation The app provides instructions before the game starts and uses meaningful buttons and helpful messages to guide the player. ✔
