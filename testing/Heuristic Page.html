User Acceptance Plan:

The web app was given to some members of team two and also to common people, for "black box testing" in order to identify and fix mistakes. The common suggestion from all people who tested the game is to improve the design when the quiz starts. Moreover, the common people that have no relation with the Uclan, cannot answer some questions. Despite of the above, they did not identify any usability issues.

Usability Report:

Heuristic Details Exist:
1. Visibility of system status Server error messages are displayed ✔
2. Match between system and the real world Universal icons e.g. qr code icon ✔
3. User control and freedom The user can skip the question if he do not know the answer ✔
4. Consistency and standards The design is based according to general treasures ✔
5. Error prevention The type of the user's respond it changes accordingly from the question ✔
6. Recognition rather than recall When the QR code got a value, the answer is sent automatically to the server ✔
7. Flexibility and efficiency of use The users can exit the game and they can continue whenever they want from the same point as they left it ✔
8. Aesthetic and minimalist design The web app does not have intensive colors or huge images ✔
9. Help users recognize, diagnose, and recover from errors If the user put a username that already exist, a warning message is displayed and the user can re-enter a new username ✔
10. Help and documentation Give instructions before the game starts, meaningful buttons and helpful messages ✔
